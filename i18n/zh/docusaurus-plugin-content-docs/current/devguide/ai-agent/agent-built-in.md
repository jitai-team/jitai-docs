# 开箱即用的内置 Agent

平台内置了多个 Agent，覆盖数据建模、数据操作和数据分析三大场景。打开聊天页面即可使用，无需任何配置。

## 需求建模 Agent

当需要根据业务描述设计数据模型时使用。Agent 会引导你澄清需求，识别核心业务对象、字段和关联关系，然后自动创建或修改数据模型。

**类型**：DeepAgent

**典型用法**：
- "我要一个订单管理系统"
- "帮我设计客户关系管理的数据模型"
- "给进销存系统建表"

## 表建模 Agent

当数据库里已经有表或视图，需要基于它们创建数据模型时使用。Agent 自动分析表/视图结构、采样数据推断字段业务语义，然后提交模型配置。

**类型**：DeepAgent

**典型用法**：
- "给这几张数据库表建模"
- "把这个视图转换成数据模型"

## 数据操作 Agent

数据模型建好后，当需要对业务数据进行增删改查时使用。Agent 自动搜索目标数据模型，按当前用户的角色权限执行查询、创建、更新或删除操作；没有权限的调用会被系统自动阻止。

**类型**：ReActAgent

**典型用法**：
- "查一下昨天的新订单"
- "把这个客户的状态改为已合作"
- "新增一条采购记录"

## 数据分析 Agent

当已有数据模型和数据后，需要深度分析业务数据、输出结构化洞察时使用。Agent 通过多角色协作（模型探索 → 查询执行 → 结果解读），输出包含数据概览、关键发现和建议行动的分析报告。

**类型**：DeepAgent

**典型用法**：
- "上个月各区域的销售趋势是什么？"
- "分析一下近三个月退货率最高的产品"
- "对比今年和去年同期的业绩"

## 如何使用

在聊天页面的 Agent 选择器中选择对应名称即可开始对话。也可以通过 URL 直接进入：

```
http://host:port/orgId/appId/AI#aiagents.DataOperationAgent
```

将 `aiagents.DataOperationAgent` 替换为其他 Agent 的 fullName 即可。

关于聊天页面的详细用法，参见[在聊天页面中使用 Agent](./agent-chat-page)。
