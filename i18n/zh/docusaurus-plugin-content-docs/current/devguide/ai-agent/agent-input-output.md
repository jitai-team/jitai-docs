# 输入输出配置

Agent 的输入输出配置决定了如何向 Agent 传递参数，以及 Agent 以什么格式返回结果。

在 Agent 编辑器中切换到`更多` → `输入 & 输出`，可以配置输入变量、输出格式和语音识别服务。

## 定义输入变量 {#configuring-input-variables}

**默认输入**是 `user_input`，即用户发送的消息内容或程序调用时传入的用户指令。

**自定义输入变量**：新增变量，定义名称、数据类型和标题。

### 变量与系统提示词的关系

输入变量定义后，可在 System Prompts 中用 `{变量名}` 引用。运行时，传入的值会自动替换提示词中的占位符：

```
系统提示词中的写法：
  你是订单查询助手。当前查询的订单编号是 {orderId}，日期范围是 {dateRange}。

运行时实际发送给模型的提示词：
  你是订单查询助手。当前查询的订单编号是 ORD20240516001，日期范围是 2024-01-01 ~ 2024-05-16。
```

变量的值来自两个途径：用户在聊天界面中预填充，或在程序调用时通过 `variables` 参数传入。

### 在聊天中使用

用户在聊天界面发送消息时，输入变量的值需要在消息之前预先填充。适合需要用户手动提供的参数（如订单号、日期范围等）。

### 在程序中使用

在服务函数或页面函数中调用 Agent 的 `run` 函数时，通过 `variables` 参数传入变量的值。详细说明请参考[将 Agent 集成到应用中](./agent-integration)。

## 定义输出格式

支持两种输出模式：

- **纯文本输出**：直接输出文本答案，适合咨询、问答和对话式回复
- **JSON 结构化输出**：按结构化 JSON 返回结果，适合需要程序化处理的场景

选择 JSON 模式后，可定义输出变量的名称、数据类型和标题，系统会自动将输出格式要求注入提示词中。

:::tip
每一个输出结果字段的标题和名称都需要仔细斟酌，做到见名知其意，以便大模型能正确理解。
:::
