# 运行时上下文注入

每次模型请求前，系统自动将当前的运行时上下文注入到提示词中，包括当前时间、用户身份和语言偏好。

## 始终注入的信息

以下内容在每次模型调用前自动注入，无需额外配置：

- **当前时间**：格式为 `YYYY-MM-DD HH:mm:ss`
- **当前用户**：用户 ID、成员 ID、昵称
- **用户语言**：根据用户输入自动检测，Agent 优先以该语言回复

## 角色与组织信息

在 `高级` → `上下文` → `运行时上下文注入` 中配置。默认不开启，支持两种互斥的获取方式：

- **智能获取**：为 Agent 提供一个工具，由模型在需要时主动获取用户角色和组织信息。适合角色信息不常使用的场景。
- **运行前注入**：每次模型请求前直接将用户角色和组织信息注入到提示词中。适合大多数交互都需要角色判断的 Agent。

两种方式均不开启时，Agent 不会感知用户角色和组织信息。

如果需要注入更复杂的业务上下文（如自定义变量、外部系统数据），可以通过[运行时扩展](./agent-runtime-extension)的 `system_prompt` 属性或 `before_model` 钩子实现。

:::info
子Agent 作为独立的 Agent 实例，可以在各自的编辑器中独立配置运行时上下文注入，互不影响。
:::
