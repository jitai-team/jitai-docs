# 配置欢迎语和预置问题

在 Agent 编辑器中切换到`更多` → `欢迎语 & 预置问题`，可配置用户打开聊天页面时的首次体验。良好的欢迎语和预置问题能让用户快速理解 Agent 的能力范围，无需思考如何提问。

欢迎语和预置问题不是装饰文字，而是 Agent 的使用入口。它们应该告诉用户“这个 Agent 能帮我完成什么”，并把用户引导到最适合测试和使用的任务。

## 欢迎语

用户打开 Agent 聊天页面时看到的第一句话。默认为"有什么我能帮你的吗？"，可根据 Agent 的定位自定义。

欢迎语应简洁说明 Agent 的身份和能力，让用户一眼就知道这个 Agent 能做什么。

建议欢迎语包含：

- Agent 的身份。
- 可处理的 2-3 类核心任务。
- 必要的输入提示，例如订单号、客户名称或日期范围。

不建议在欢迎语中承诺 Agent 不能稳定完成的能力，也不要写过长的说明。

## 预置问题

预置问题展示在欢迎语下方，用户点击即可快速发起对话，无需手动输入。支持添加多个预置问题，每个问题都可随时修改或删除。

预置问题应覆盖用户最可能提出的典型需求，引导用户快速上手。

好的预置问题通常具备三个特点：

- 能触发 Agent 的核心能力。
- 输入条件清晰，便于用户替换为自己的业务对象。
- 覆盖查询、分析、生成、处理等不同任务类型。

## 典型配置示例

- **订单助手**
  - 欢迎语：`你好，我是订单助手，可以帮你查询、修改和取消订单。`
  - 预置问题：`查询我的最新订单`、`取消最近一笔订单`、`修改收货地址`

- **数据分析 Agent**
  - 欢迎语：`你好，我是数据分析助手，可以帮你分析销售数据和生成报表。`
  - 预置问题：`本月销售趋势如何？`、`按地区分析客户分布`、`对比上月和本月销售额`

## 用预置问题验证 Agent 能力边界

预置问题也可以作为上线前测试用例。建议至少准备：

- 一个最常用的正常任务。
- 一个需要补充信息的任务。
- 一个会触发知识库、工具或 Skill 的任务。
- 一个高风险或超出范围的问题，用来验证 Agent 是否会受权限拦截或拒绝处理。

如果用户点击预置问题后经常得不到稳定结果，优先检查 Agent 的工具配置、输入输出配置和 System Prompts，而不是继续增加更多预置问题。
