# 用组织空间维护团队共享资料

组织空间用于保存团队共享文件，例如制度文档、产品资料、公共模板、操作手册和标准案例。多个 Agent 和多个用户可以读取同一个组织空间。

组织空间默认只读，避免 Agent 误修改团队资料。只有在 Agent 的文件空间配置中为指定组织空间开启编辑权限后，该 Agent 才能写入。

## 从 AI 文件空间入口创建组织空间 {#create-org-space}

进入 IDE 后，在元素目录树中点击 `+`，选择 `AI 文件空间` 下的 `组织空间`。

创建时填写文件空间名称和根目录。保存后，该组织空间会出现在管理区门户的组织文件空间管理页面中，也可以被 Agent 选择使用。

## 组织空间适合保存什么 {#what-belongs-in-org-space}

适合放在组织空间的文件：

- 公司制度、流程说明、操作手册。
- 产品资料、价格表、服务说明。
- 团队共享模板和示例文件。
- 多个 Agent 都会读取的公共资料。

不适合放在组织空间的文件：

- 用户个人上传文件。
- 某个 Agent 每次运行产生的临时文件。
- 不希望被多个 Agent 或多个用户共享的敏感文件。

如果资料需要被检索问答频繁使用，也可以考虑同步到[知识库](../knowledge-base/create-knowledge-elements#create-ai-knowledge-base-element)。组织空间更适合维护原始文件和可下载资料。

## 组织空间默认只读 {#org-space-is-readonly-by-default}

组织空间默认不允许 Agent 写入。这样可以避免 Agent 在执行任务时误删、误改团队公共资料。

如果确实需要某个 Agent 更新组织空间，例如生成团队周报、维护公共模板或同步外部资料，需要在该 Agent 的文件空间配置中为对应组织空间开启编辑权限。

建议只对少数经过验证的 Agent 开启组织空间写入权限，并结合[控制 Agent 访问权限](../ai-agent/agent-permissions)控制可使用人群。

## 组织空间不自动清理 {#org-space-no-auto-cleanup}

组织空间用于长期维护团队资料，不适合自动按时间清理。文件删除、替换和整理应由有管理门户权限的用户在管理区门户的组织文件空间管理页面中手动处理。

管理方法见[管理组织空间中的共享文件](./manage-org-files)。
