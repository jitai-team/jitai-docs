# 为每个用户准备私有文件空间

个人空间用于保存当前登录用户自己的上传文件和生成文件。它固定按用户隔离，不同用户看不到彼此的文件。

当用户会在聊天中上传文件，或希望跨 Agent、跨会话保留自己的分析资料、报告和草稿时，适合使用个人空间。

## 从 AI 文件空间入口创建个人空间 {#create-user-space}

进入 IDE 后，在元素目录树中点击 `+`，选择 `AI 文件空间` 下的 `个人空间`。

创建后，可以在 Agent 的文件空间配置中选择这个个人空间。配置完成后，用户上传的文件可以被 Agent 通过个人空间路径访问。

## 个人空间适合保存什么 {#what-belongs-in-user-space}

适合放在个人空间的文件：

- 用户在聊天中上传的 Excel、Word、PDF、图片等文件。
- Agent 针对当前用户生成的分析报告。
- 用户自己的草稿、临时资料和处理结果。
- 需要跨会话继续使用的个人文件。

不适合放在个人空间的文件：

- 团队共享制度、产品资料和公共模板。
- 需要多个用户共同维护的资料。
- 某个 Agent 内部长期复用的工作文件。

团队共享资料更适合放在[组织空间](./create-org-space)，Agent 自己的工作文件更适合放在[Agent 空间](./create-agent-space)。

## 用户上传文件如何进入个人空间 {#uploaded-files-go-to-user-space}

当 Agent 配置了个人空间后，用户在聊天页面上传的文件会保存到个人空间中。Agent 可以看到上传文件的逻辑路径，并使用文件工具读取。

常见路径类似：

```text
/User/uploads/销售数据.xlsx
```

其中 `User` 是个人空间的文件空间名称。实际名称以创建个人空间时填写的名称为准。

## 清理策略要根据业务保留周期设置 {#cleanup-policy-for-user-files}

个人空间可能保存用户长期需要的文件，也可能只保存临时上传资料。建议按业务场景设置清理策略：

- 只处理临时上传文件：可以开启清理，并设置较短阈值。
- 保存用户长期报告和草稿：应谨慎开启清理，或设置较长阈值。
- 涉及合规留存的文件：不要依赖自动清理，应按组织的数据保留要求配置。
