# 让 Agent 有地方保存、读取和交付文件

AI 文件空间是 Agent 可访问的文件存放区域。它解决的是：用户上传的文件放在哪里、Agent 生成的报告保存在哪里、团队共享资料如何让 Agent 读取、不同用户和不同 Agent 的文件如何隔离。

当 Agent 需要读取文件、处理上传资料、生成报告、保存中间结果或查阅组织资料时，就需要配置 AI 文件空间。

## 三类文件空间分别解决什么问题 {#three-types-of-file-spaces}

平台提供三类 AI 文件空间：

| 类型 | 主要用途 | 隔离方式 | 典型场景 |
| --- | --- | --- | --- |
| **Agent 空间** | 保存某个 Agent 的工作文件、临时结果、生成文件 | 按 Agent 隔离，可选继续按用户隔离 | 报告生成、数据分析结果、任务过程文件 |
| **个人空间** | 保存当前用户自己的上传文件和生成文件 | 固定按用户隔离 | 用户上传 Excel、个人分析结果、跨 Agent 个人资料 |
| **组织空间** | 保存团队共享资料 | 多 Agent、多用户共享，默认只读 | 公司制度、产品资料、公共模板、团队知识文件 |

一个 Agent 可以同时配置 Agent 空间、个人空间和多个组织空间。不同空间承担不同职责，能减少文件混放和权限误用。

## 文件空间、知识库和 Skill 如何分工 {#file-space-knowledge-base-skill}

AI 文件空间、知识库和 Skill 都能辅助 Agent，但职责不同：

| 能力 | 适合存放或表达什么 |
| --- | --- |
| **AI 文件空间** | 可读写文件、上传资料、生成报告、团队共享文件 |
| **知识库** | 面向检索问答的结构化知识和文档索引 |
| **Skill** | 可复用的业务流程、判断规则、输出规范和示例 |

例如“合同审查 Agent”可以这样组合：

- 组织空间存放合同模板和审查清单文件。
- 知识库存放法规、制度和历史问答资料。
- Skill 规定审查步骤、风险等级和输出格式。
- Agent 空间保存每次审查生成的报告。

## 什么时候应该先创建文件空间 {#when-to-create-file-space}

以下场景建议先创建文件空间，再配置 Agent：

- 用户会在聊天中上传文件，Agent 需要读取并分析。
- Agent 会生成报告、表格、Markdown、JSON 或其它结果文件。
- 多轮任务需要保留中间文件，后续继续使用。
- Agent 需要读取团队共享资料，但这些资料不适合做成知识库。
- 组织资料需要人工上传、替换、删除和整理。

如果 Agent 只需要查询数据库、调用服务函数或回答知识库问题，不一定需要文件空间。

## 推荐的创建顺序 {#recommended-setup-order}

建议按下面顺序配置：

1. 判断 Agent 是否需要读写文件。
2. 选择需要的文件空间类型。
3. 创建 Agent 空间、个人空间或组织空间元素。
4. 在目标 Agent 的文件空间配置中选择这些空间。
5. 在 System Prompts 中说明文件路径规则。
6. 用真实上传文件和生成文件任务验证效果。

如果你已经在配置 Agent，可以直接阅读[在 Agent 中配置可访问的文件空间](./use-file-space-in-agent)。
