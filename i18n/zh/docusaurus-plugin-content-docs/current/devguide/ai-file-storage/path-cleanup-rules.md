# 规划稳定的文件路径、隔离和清理规则

文件空间创建后，最容易出问题的地方通常不是“能不能保存文件”，而是文件路径不稳定、空间职责混乱、清理策略不合适。建议在上线前统一规划文件空间名称、目录结构、隔离方式和清理周期。

## 文件空间名称就是路径前缀 {#space-name-is-path-prefix}

Agent 访问文件时使用逻辑路径：

```text
/{spaceName}/{相对路径}
```

例如：

```text
/Workspace/report.md
/User/uploads/sales.xlsx
/OrgDocs/policies/security.md
```

其中 `Workspace`、`User`、`OrgDocs` 就是文件空间名称。

命名建议：

- 使用英文、数字、下划线或中划线。
- 以字母开头。
- 保持短且稳定。
- 同一个 Agent 中不要重复。

## 根目录不是 Agent 使用的路径 {#root-dir-is-not-agent-path}

根目录是平台保存文件的物理位置。Agent 不应该看到或使用根目录。

开发者在创建文件空间时填写根目录，Agent 在对话和工具调用中只使用逻辑路径。不要在 System Prompts 中写服务器路径，也不要让用户输入服务器目录。

## 三类空间的隔离规则不同 {#isolation-rules}

| 类型 | 隔离规则 | 说明 |
| --- | --- | --- |
| Agent 空间 | 按 Agent 隔离，可选按用户隔离 | 适合 Agent 工作文件 |
| 个人空间 | 固定按用户隔离 | 适合用户上传和个人结果 |
| 组织空间 | 不按 Agent 或用户隔离 | 适合团队共享资料 |

如果文件包含用户隐私或个人业务数据，优先使用个人空间，或在 Agent 空间中开启按用户隔离。

## 清理策略适合工作文件，不适合组织资料 {#cleanup-policy}

Agent 空间和个人空间可以配置清理策略。开启后，平台会按阈值清理长时间未更新的文件。

适合开启清理的场景：

- 临时分析结果。
- 中间过程文件。
- 用户上传的短期处理资料。
- 测试和调试文件。

不适合短期清理的场景：

- 用户需要长期保留的报告。
- 团队共享资料。
- 合规要求保留的文件。
- 被提示词或 Skill 固定引用的模板文件。

组织空间用于长期资料维护，不自动清理，需要由有管理门户权限的用户在管理区门户的组织文件空间管理页面中手动整理。

## 上线前检查清单 {#release-checklist}

上线前建议确认：

- 每个文件空间名称都清晰且不重复。
- System Prompts 中使用的是逻辑路径，不是根目录。
- 用户上传文件能进入个人空间并被 Agent 读取。
- Agent 生成文件能落到预期空间。
- 组织空间默认只读，只有必要 Agent 开启写入。
- 清理策略不会误删需要长期保留的文件。
- 高风险写入操作有角色权限控制，非授权调用会被运行时自动阻止。
