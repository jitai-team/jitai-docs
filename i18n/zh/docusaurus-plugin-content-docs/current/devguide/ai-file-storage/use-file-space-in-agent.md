# 在 Agent 中配置可访问的文件空间

创建 AI 文件空间后，还需要在 Agent 中选择使用，Agent 才能读写对应空间中的文件。

在 Agent 编辑器中切换到 `更多` → `文件空间`，可以配置当前 Agent 可访问的 Agent 空间、个人空间和组织空间。

## 选择 Agent 空间保存工作文件 {#select-agent-space}

Agent 空间用于保存当前 Agent 的工作文件和生成结果。适合：

- 数据分析过程文件。
- 生成的报告、表格、图表配置。
- 长任务中的中间结果。
- 当前 Agent 专属的任务资料。

如果 Agent 不需要保存任何工作文件，可以清空 Agent 空间配置。

## 选择个人空间处理用户上传文件 {#select-user-space}

个人空间用于处理当前用户自己的上传文件和生成文件。适合：

- 用户上传文件后让 Agent 读取分析。
- Agent 把处理结果保存给当前用户后续继续使用。
- 同一个用户跨多个 Agent 共享自己的文件。

未配置个人空间时，用户上传文件无法被 Agent 通过文件工具稳定访问。

## 选择组织空间读取团队资料 {#select-org-space}

组织空间用于让 Agent 读取团队共享资料。一个 Agent 可以选择多个组织空间。

组织空间默认只读。只有当 Agent 确实需要维护组织资料时，才为对应组织空间开启编辑权限。

建议：

- 客服、销售、问答类 Agent：通常只读组织空间。
- 报表归档、资料同步类 Agent：可以为指定组织空间开启写入，但要限制可使用人群。
- 任何会删除或替换组织文件的 Agent：必须结合角色权限、业务复核流程和运行时权限拦截。

## 在提示词中明确文件路径规则 {#write-path-rules-in-prompts}

Agent 使用的是逻辑路径，格式为：

```text
/{文件空间名称}/{相对路径}
```

建议在 System Prompts 中写清楚：

```markdown
文件路径规则：
- 工作文件写入 /Workspace/
- 用户上传文件从 /User/uploads/ 读取
- 团队资料从 /OrgDocs/ 读取
- 不要使用服务器物理目录
```

这样可以减少 Agent 把文件写到错误空间的概率。

## 用真实文件任务验证配置 {#verify-with-real-file-tasks}

配置完成后，建议用真实任务验证：

- 上传一个文件，确认 Agent 能读取。
- 让 Agent 生成一个报告，确认聊天页面出现可下载文件。
- 让 Agent 读取组织资料，确认路径和权限正确。
- 如果开启了组织空间写入，测试是否只在预期目录写入。

Agent 侧的文件工具、上传路由和运行效果说明见 [Agent 文件空间](../ai-agent/agent-file-space)。
